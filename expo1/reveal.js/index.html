Cross Site Scripting

XSS

Es una vulnerabilidad de seguridad que permite a un atacante inyectar en un sitio web código malicioso del lado del cliente.
Este código es ejecutado por las vícitmas y permite a los atacante eludir los controles de acceso y hacerce pasar por usuarios.

Los ataques XSS suelen producirse cuando:

  • Los datos entran en una aplicación web no confiable
  • El contenido dinámico se envía a un usuario web sin ser validado como contenido malicioso

XSS Reflejado

Este tipo de ataque se caracteriza por no almacenar en el servidor el código utilizado. En su lugar el código malicioso forma parte de la petición de la víctima que accede a la web. Esta web incluye el código en la respuesta a la petición de la víctima, que se ejecuta en el destino

XSS Persistente

Se da cuando el script se queda almacenado en el servidor. Cualquier usuario que acceda a la página no sospechará nada, sin embargo, el código será interpretado por el navegador del visitante, ejecutando las instrucciones que el atacante haya definido.
